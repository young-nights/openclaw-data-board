## 目标
在 OpenClaw 的 Web Control UI / Mission Control 中创建一个**视觉和交互高度一致**的 Activity 仪表盘，用于监控 OpenRouter（或其他兼容 API）的使用情况：花费、请求数、Token 消耗。

页面必须实现“**总览卡片 → 点击放大钻取**”的交互模式。

## 整体页面布局要求（严格遵守）

1. **顶部区域**
   - 页面标题：`Activity` 或 `OpenRouter Activity`
   - 右侧控制栏：
     - 时间范围选择器（1 Hour / 1 Day / 7 Days / 1 Month / 1 Year / Custom）
     - 分组维度选择（By Model / By API Key / By Provider）
     - 导出按钮（CSV + PDF）

2. **核心区域：三张大指标卡片（横向排列或响应式卡片）**
   - **Spend Card**（花费）
     - 主显示：Total Spend（$X.XX）+ OpenRouter Credits
     - 子信息：今日 / 本月 / 预估
   - **Requests Card**（请求数）
     - 主显示：Total Requests（含 Chatroom）
   - **Tokens Card**（Token 用量）
     - 主显示：Total Tokens（Prompt + Completion + Reasoning + Cached）

   **卡片样式要求**：
   - 大尺寸、圆角、阴影、醒目数值
   - 支持颜色区分（Spend → 橙/黄系，Requests → 绿系，Tokens → 青/绿系）
   - **必须支持点击**：点击任意卡片 → 在当前位置或模态中“放大”显示该指标的详细视图

3. **详细视图（点击后展开/放大面板）统一结构**
   每个详细视图必须包含：
   - **上方**：时间趋势柱状图（Bar Chart）
     - X 轴：时间（根据时间范围自动调整粒度）
     - Y 轴：对应指标（金额 / 请求数 / Token 数）
     - 不同模型用不同颜色柱子 + 图例
   - **下方**：按模型（或分组）排序的详细表格
     - 列（示例，以 Spend 为准）：
       - Model（带颜色点）
       - Min ($)
       - Max ($)
       - Avg ($)
       - Sum ($) ← 最重要列
     - 支持列点击排序（↑↓）
   - **右上角**：额外过滤器 + 导出菜单（CSV / PDF）
     - Tokens 视图额外支持切换：Total / Prompt / Reasoning / Completion / Cached

4. **底部通用元素**
   - “Logs have moved” 提示（指向独立 Logs 页面）
   - 刷新按钮 + 最后更新时间

## 数据来源建议（OpenClaw 实现方式）
- 优先通过 OpenRouter 官方 `/api/v1/activity` 或 `/generation` 接口拉取数据
- 也支持本地日志解析（如果有保存 OpenRouter 请求记录）
- 支持缓存 + 定时刷新（每 5~15 分钟）
- 如果数据不足，优雅降级显示“暂无数据”或模拟数据

## UI 组件约束（推荐使用 OpenClaw A2UI）
- 使用 `<Card>` 包裹每张主卡片和详细面板
- 使用 `<Chart>` 组件渲染柱状图（支持颜色、图例、hover 提示）
- 使用 `<Table>` 渲染详细数据，支持排序
- 交互：点击卡片触发 `showDetail(viewType)`（viewType = spend | requests | tokens）
- 响应式：手机/桌面均良好显示，三卡片在小屏时可垂直排列

## Agent 行为约束
- 当用户说“打开 OpenRouter Activity”、“看花费”、“查看 Token 使用”时，自动打开此仪表盘。
- 支持自然语言查询，例如：
  - “这个月哪个模型花钱最多？”
  - “显示 MiMo-V2-Pro 的 Token 详情”
  - “导出上周的花费 CSV”
- 详细视图中允许进一步过滤和导出。
- 保持界面干净、专业，与 OpenRouter 原页面风格接近（简洁、数据优先、现代卡片设计）。

## 扩展性
- 未来可扩展支持其他提供商（Groq、Together、Fireworks 等）的类似 Activity 视图。
- 可添加“Cost Alert” 阈值提醒（花费超过 $X 时高亮或通知）。
- 支持多 API Key 分组查看。

## 实现提示（给 Coding Agent）
- 先实现三张汇总卡片 + 点击事件。
- 再实现详细钻取面板（复用图表和表格组件）。
- 使用 Tailwind / A2UI 组件库保持一致风格。
- 确保数据加载有 loading 状态和错误处理。

此文档定义了**精确的视觉布局、交互流程和数据展示要求**。Agent 在构建或更新 Dashboard 时必须严格遵循以上规范，以达到接近原 OpenRouter Activity 页面的用户体验。
