# OpenClaw UI 重构优化约束条件文档

**项目名称**：openclaw-data-board（OpenClaw 控制中心）  
**重构目标**：将传统 HTML + SSE 轮询面板重构为 **LVGL 风格的丝滑现代 UI**  
**文档版本**：v1.0  
**更新日期**：2026-04-05  
**开发者**：Young Nights

## 1. 总体目标

- 操作体验必须达到 **LVGL 嵌入式界面级别** 的丝滑度（无闪烁、无延迟、动画自然、实时数据局部更新）。
- 最终实现前后端完全分离，前端使用现代框架，后端仅提供纯 API + 静态文件服务。
- 性能目标：生产环境下前端 bundle（gzip 后）≤ 200KB，首屏加载时间 < 800ms。
- 必须同时支持 Windows 开发环境和 Ubuntu 生产环境。

## 2. 技术栈约束（必须严格遵守）

| 类别         | 必须使用技术                          | 禁止使用                          | 说明 |
|--------------|---------------------------------------|-----------------------------------|------|
| 前端框架     | Svelte 5（Runes + $state / $derived） | React、Vue、Angular               | 追求极致轻量与响应式 |
| 构建工具     | Vite 6+                               | Webpack、Parcel                   | 必须使用 Vite |
| 后端服务     | Node.js 原生 http + serve-static      | Express、Koa（除非必要）          | 保持轻量 |
| 编程语言     | TypeScript（严格模式）                | 纯 JavaScript                     | 全项目使用 TS |
| 样式方案     | 原生 CSS + CSS Variables + Glassmorphism | Tailwind（暂不允许）             | 保留现有视觉风格 |
| 实时通信     | SSE（优先）或 WebSocket               | 传统轮询（长轮询禁止）            | 必须实现 delta 更新 |
| 动画实现     | CSS transition + will-change + rAF    | jQuery、GSAP（除非必要）          | 追求 GPU 加速 |

## 3. 项目架构约束

- 必须实现**前后端分离**：
  - 新建 `frontend/` 目录作为独立 Vite + Svelte 项目
  - 后端 `src/ui/server.ts` 只负责提供 API 接口和托管 `dist/` 静态文件
- 所有数据更新必须使用**细粒度响应式**，禁止全页面刷新或全组件重渲染
- 端口管理必须使用环境变量 `UI_PORT`，默认推荐 `4330`
- 前端构建输出目录必须为项目根目录下的 `dist/`

## 4. 代码与文件规范

- **文件名与路径**：全部使用小写字母 + 连字符（kebab-case）
- **行尾符**：必须统一为 LF（在 VSCode 中设置为 LF）
- **import 路径**：推荐使用相对路径或 alias，禁止大小写混用
- **必须清理的文件**（重构完成后应删除）：
  - 旧的 HTML 字符串模板渲染逻辑
  - 所有 inline `<style>` 和 `<script>`
  - `package-lock.json`、`ecosystem.config.cjs`
  - `docs/`、`test/`、`scripts/ui-smoke.js` 等非核心目录和文件

## 5. 性能与用户体验约束（LVGL 风格核心要求）

- 所有卡片、仪表盘、列表的出现/更新必须带有平滑过渡动画（duration 200~400ms）
- 数据更新必须使用 `$state` / `$derived`，禁止不必要的组件重新渲染
- 仪表盘（Gauge）建议使用 Canvas + `requestAnimationFrame` 实现平滑动画
- 按钮点击必须有 `active:scale-95` 等微交互反馈
- 长列表应考虑虚拟滚动
- 禁止任何会导致页面闪烁、白屏或卡顿的操作

## 6. 跨平台兼容约束

- 所有路径必须使用 `path.join()` 或 `/`
- 项目中不得出现 Windows 专有命令
- 必须在项目根目录放置 `.npmrc` 文件（配置 store-dir、shamefully-hoist 等）
- 必须包含 `.gitattributes` 文件强制使用 LF
- 生产部署时使用 `pnpm install --production`

## 7. 开发流程约束

- 日常开发命令：`pnpm dev`（必须同时启动 API 和 Frontend）
- 必须支持 VSCode 完整调试（Svelte 组件 + Node.js 后端）
- 每次重大修改后必须验证：
  - 前端热更新正常工作
  - 后端 API 正常返回
  - 无端口冲突
  - Windows 与 Ubuntu 路径兼容

## 8. 禁止事项（红线）

- 禁止在前端使用任何 `location.reload()` 或全页刷新
- 禁止继续使用旧的字符串模板渲染 HTML
- 禁止在 Svelte 组件中大量使用 inline style（应统一使用 CSS）
- 禁止提交 `node_modules`、`dist`、`.vite` 等文件夹
- 禁止使用任何会阻塞主线程的操作
